01

Fade-up 入場

九成內容的工作馬 · 林事／參詳這類工具型網站的預設鍵
載入▸opacity＋translateY 20px▸700ms▸ease-out▸stagger 90ms
CSS 
@keyframes fadeUp {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}
.fadeup { opacity: 0; }
.fadeup.run {
  animation: fadeUp .7s cubic-bezier(0.16, 1, 0.3, 1) both;
}
/* 子元素魚貫進場：每個晚 90ms */
.fadeup:nth-child(2).run { animation-delay: 90ms; }
.fadeup:nth-child(3).run { animation-delay: 180ms; }
.fadeup:nth-child(4).run { animation-delay: 270ms; }
02

緩動對照

同參數只換曲線 · 感受「質感」到底藏在哪
ease-out有人把它放下
linear輸送帶
spring有生命
載入▸opacity＋translateY 86px▸900ms▸三種曲線▸無 stagger
CSS 
/* 質感三兄弟：入場請用前者，linear 只配跑馬燈 */
--easeOut: cubic-bezier(0.16, 1, 0.3, 1);   /* 急起緩停 */
--spring:  cubic-bezier(0.34, 1.56, 0.64, 1); /* 過衝回彈 */
/* linear：等速，入場用它 = 廉價感頭號來源 */

@keyframes rise {
  from { opacity: 0; transform: translateY(86px); }
  to   { opacity: 1; transform: translateY(0); }
}
.rise.run { animation: rise .9s var(--easeOut) both; }
03

霧化入場 Blur-in

hero／攝影作品 · 單一元素限定（filter 重繪很貴，列表勿用）
載入▸opacity＋blur 16px→0＋scale 1.05→1▸900ms▸ease-out▸單元素
CSS 
@keyframes blurIn {
  from { opacity: 0; filter: blur(16px); transform: scale(1.05); }
  to   { opacity: 1; filter: blur(0);   transform: scale(1); }
}
.blurin { opacity: 0; }
.blurin.run {
  animation: blurIn .9s cubic-bezier(0.16, 1, 0.3, 1) both;
}
/* ⚠ filter 觸發重繪：只給 hero 這種單一大元素，
   低階手機的長列表用了會掉幀 */
04

Spring 彈跳

活動頁的張力 · badge／CTA／潮界帶這種場合
潮界帶 海邊的客廳 LÜCY · LIVE
載入▸opacity＋scale 0.8→1＋translateY 10px▸600ms▸spring▸stagger 60ms
CSS 
@keyframes pop {
  from { opacity: 0; transform: scale(.8) translateY(10px); }
  to   { opacity: 1; transform: scale(1) translateY(0); }
}
.pop { opacity: 0; }
.pop.run {
  animation: pop .6s cubic-bezier(0.34, 1.56, 0.64, 1) both;
}
.pop:nth-child(2).run { animation-delay: 60ms; }
.pop:nth-child(3).run { animation-delay: 120ms; }
/* 註：CSS 的 spring 是 cubic-bezier 假裝的「單次回彈」，
   要真物理彈簧（可打斷、多次震盪）才需要 Framer Motion */
05

Hover 微反饋

150–250ms 區 · 卡片／按鈕的「活著」訊號
回報樹況3 分鐘完成
物種百科127 筆
活動場次7/4 出張所

hover 我（觸控裝置沒有 hover，行動端要另給 :active 態）

hover▸translateY −3px＋shadow＋border▸200ms▸ease-out▸回程共用同曲線
CSS 
.hcard {
  transition: transform .2s cubic-bezier(0.16, 1, 0.3, 1),
              box-shadow .2s cubic-bezier(0.16, 1, 0.3, 1),
              border-color .2s cubic-bezier(0.16, 1, 0.3, 1);
}
.hcard:hover {
  transform: translateY(-3px);
  box-shadow: 0 10px 24px rgba(0, 0, 0, .38);
  border-color: var(--accent);
}
/* transition 寫在基底而非 :hover 上，
   滑開時的「回程」才會一樣順 */
06

捲動觸發 Reveal

長頁內容區的標配 · IntersectionObserver 進場一次
§1法規查詢
§2合規檢查
§3會議提醒
§4樹況回報
§5減廢儀表板
§6德菲法指標
— 在框內往下捲 · 每列進視口才現身 —
進入視口 35%▸opacity＋translateY 24px▸700ms▸ease-out▸只演一次
CSS ＋ JS 
.reveal {
  opacity: 0; transform: translateY(24px);
  transition: opacity .7s cubic-bezier(0.16, 1, 0.3, 1),
              transform .7s cubic-bezier(0.16, 1, 0.3, 1);
}
.reveal.in { opacity: 1; transform: none; }

// JS：進視口 35% 就演，演完退訂（只演一次）
const io = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.classList.add('in');
      io.unobserve(e.target);
    }
  });
}, { threshold: 0.35 }); // 頁面級用法不需指定 root
document.querySelectorAll('.reveal').forEach(el => io.observe(el));

守則：一頁最多 一套入場＋一套 hover，超過就開始吵。正式環境務必尊重使用者的減少動態偏好——

prefers-reduced-motion 全域片段（直接貼進 globals.css） 
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
