# 學生會法規整合網站 — 專案啟動 Prompt

## 背景與目標
手上有一批學生會相關法規（附件），想做一個網站，達成三個核心功能：

1. **法規查詢**：之後要查法規方便查、方便讀
2. **合法性檢核**：丟一份文件或提案進去，對照現有法規，判斷是否合法／符合規定
3. **時程管理與提醒**：法規裡規定的會議時程（議會、委員會多久要開一次、幾日內須召開等）能被整理出來、排進行事曆並提醒

## 第一階段：先規劃，不要動手寫 code

### Step 1 — 看懂法規本身
先讀過附件，抓出：
- 整體章節結構（章/條/款/項的階層）
- 是否有版本／修正歷程需要處理（法規會不會被修正、要不要保留舊版本）
- 條文之間的交互參照（例如「依第 X 條規定辦理」這類引用）
- 哪些條文屬於「時程／期限」類型（開會頻率、幾日內須召開等），適合抽取成結構化資料

### Step 2 — 針對三個功能個別提出架構草案

**法規查詢**
- 資料表設計
- 全文搜尋策略（可參考參詳專案用 tsvector + GIN 的做法；如果你評估這個情境更適合語意搜尋或混合搜尋，直接講理由）

**合法性檢核**
這個功能等於「上傳文件 → 找出相關法規 → 判斷是否合規」，麻煩你想清楚：
- 相關法規怎麼檢索（RAG 的 retrieval 策略）
- 判斷結果怎麼呈現——不要只回「合法/不合法」，要附上引用的具體條文、可能的疑慮點、信心程度
- 這個功能背後呼叫的 LLM 建議用哪個 model（要考慮判斷準確度 vs. 之後每次檢核的成本）

**時程管理與提醒**
- 法規抽出的時程規則怎麼存
- 怎麼跟提醒機制串起來（站內提醒、或串 Google Calendar）

### Step 3 — 給我一個實作順序建議
我的直覺是法規查詢是地基，其他兩個功能都要建立在這個資料庫上，但你可以挑戰這個假設，直接告訴我你覺得合理的順序。

## 技術慣例
沿用「網站建立流」的技術棧：Next.js 14 App Router + TypeScript + Tailwind + Zeabur PostgreSQL，有檔案儲存需求就用 Cloudflare R2。
